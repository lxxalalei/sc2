# 资源来源选择策略

## 核心原则

学习资源请求不绑定单一来源。任何 source skill 都只是候选来源之一。

来源选择保持类型无关、主题无关、格式无关、年龄阶段无关和使用场景无关。应根据用户需求生成一个或多个来源任务，最后由 `learning-resource-ranker` 和 `learning-resource-selector` 按质量、权威性、适配度和可访问性排序展示。

## 来源角色

- 精确来源 skill：适合结构清晰、接口明确、来源稳定的资源站点。
- 通用标准化 skill：适合把 agent 已获得的搜索结果转换为统一候选格式。
- 本地检索 skill：适合查询已有资料库。
- 后续新增来源应作为候选来源加入同一流程，而不是改变全局规则。

## 来源任务生成

生成 `execution_tasks` 时：

1. 先判断用户需求是否足够明确。
2. 根据来源可用性、用户偏好和资源目标生成一个或多个候选来源任务。
3. 优先查询已经优化好的站点 source skills，并读取其 source profile 判断能力边界。
4. 已优化来源候选不足、质量不足或覆盖不完整时，再进入 agent 通用网络搜索。
5. 通用搜索结果先标准化，再做来源发现、站点画像和必要的通用抽取。
6. 来源任务只负责发现候选，不负责决定最终推荐。
7. 多个来源返回的候选统一交给 analyzer、ranker、selector。
8. 权威来源可在评分阶段加权，但不应跳过候选展示和用户选择。

## 设计准则

- source skill 只负责发现候选。
- 当前测试源和已实现来源只代表可用候选来源。
- 新增来源时只扩展来源集合，不改写全局意图理解规则。
- source-first 不等于固定来源；它只表示“先用已掌控的高确定性来源”，再按阈值和质量进入更开放的网络搜索。
- 最终推荐由 ranker 和 selector 完成。
